# 任务目录

本仓库里，**每个任务都有自己的独立文件夹**（`tasks/<task-name>/`）。  
任务可能随时成熟并**转正迁出**为独立项目；这里只是临时收容，不是终点。

> 一任务一目录 · 随时可能转正

---

## 约定

| 规则 | 说明 |
|------|------|
| **一任务一目录** | 相关脚本、配置、笔记都放进该任务文件夹，不散落混放 |
| **可转正** | 被频繁使用、需要 CI / issue / release 时，迁出成独立 repo |
| **可追溯** | 在下方清单登记名称、路径、状态；转正后保留一行并链到新项目 |

### 新建任务

```bash
mkdir -p tasks/<task-name>
# 在该目录内放脚本、配置、README/笔记等
```

在下方「当前任务」表加一行，状态标为 `进行中`。

### 转正迁出

1. 创建独立 repo，迁入内容  
2. 本表将该行状态改为 `已转正`，并填上新项目链接  
3. 原目录可挪到 `archived/` 或删除（按需保留参考）
